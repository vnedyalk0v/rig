# Support

`rig` is pre-release. The current source of truth is
[docs/rig-v1-spec.md](docs/rig-v1-spec.md).

## Questions

Open a GitHub issue for non-sensitive questions about the v1 design, planned
behavior, or contribution direction.

## Bugs

Use the bug report template and include the smallest reproduction you can.

## Feature Requests

Use the feature request template and explain how the request fits the v1 scope.

## Security Issues

Do not report vulnerabilities in public issues. Follow
[SECURITY.md](SECURITY.md).
